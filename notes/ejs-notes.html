01
What is EJS?
A templating engine — HTML that knows about your data
Key Concept

EJS (Embedded JavaScript) is a templating engine for Node.js. It lets you write HTML files that contain JavaScript expressions inside special <% %> tags. When a request comes in, Express runs the template, fills in all the dynamic parts with real data, and sends back a complete finished HTML page to the browser.

Think of EJS like a mail merge. You write one template letter with placeholders like "Dear [NAME]". For each recipient, the placeholders get filled in and a unique letter is printed. EJS does the same — one template, infinite dynamic pages.

What EJS is good for

Use Case
Admin dashboards

Internal tools where you just need to display data — users table, order list, metrics. No need for a full React app.

Use Case
Email templates

Generate personalised HTML emails server-side. "Hi Harshit, your order #4521 is confirmed." EJS fills the values in before sending.

Use Case
Multi-page websites

Blogs, portfolios, marketing sites. Each page is rendered fresh with data from the DB — no API, no JSON, no fetch calls.

Use Case
SEO-critical pages

HTML arrives fully rendered from the server. Search engine crawlers read actual content immediately — no waiting for JS to run.

02
How Server-Side Rendering Works
The flow from browser request to finished HTML page
Browser sends a request
User visits /dashboard. A GET request hits your Express route.
Express fetches data
Your route handler queries MongoDB, gets the user's data, orders, stats — whatever that page needs.
Express renders the EJS template
Calls res.render('dashboard', { user, orders }). EJS takes the template and the data, runs all the <% %> tags, and produces a plain HTML string.
Server sends finished HTML
The browser receives a complete HTML page — no JavaScript needed to render content. It paints instantly. ✅
CSR (React SPA):
 Browser gets blank HTML → downloads JS bundle → JS runs → fetches API → renders page
 User sees blank screen for 1–3 seconds. Bad for SEO.

SSR (EJS):
 Browser gets complete HTML with data already in it
 User sees content immediately. Great for SEO. ✅
03
Setup with Express
Install, configure the view engine, and set up the folder structure
Step 1 — Install EJS
Terminal
npm install ejs
Step 2 — Tell Express to use EJS
app.js
import express from 'express';
import path    from 'path';
import { fileURLToPath } from 'url';

const app = express();
const __dirname = path.dirname(fileURLToPath(import.meta.url));

// Set EJS as the view engine
app.set('view engine', 'ejs');

// Tell Express where to find the template files
app.set('views', path.join(__dirname, 'views'));

// Serve static files (CSS, images, JS)
app.use(express.static(path.join(__dirname, 'public')));
Folder Structure
project layout
project/
├── views/                  ← all .ejs template files live here
│   ├── index.ejs
│   ├── dashboard.ejs
│   └── partials/
│       ├── header.ejs
│       └── footer.ejs
├── public/                 ← static assets (CSS, images, JS)
│   ├── style.css
│   └── app.js
├── app.js
└── routes/
Step 3 — Render a view from a route
routes/index.js
router.get('/', async (req, res) => {
  const products = await Product.find();

  // res.render('filename', { data to pass to template })
  res.render('index', {
    title: 'Home',
    products,
    user: req.user || null
  });
});
💡

res.render('index') automatically looks for views/index.ejs. You never write the .ejs extension or the views/ path — Express handles that for you.

04
EJS Tags
Five tag types — each does something different
Output (escaped)
<%= value %>
Outputs the value as HTML. Automatically escapes < > & to prevent XSS. Use this for user data.
Execute (no output)
<% code %>
Runs JavaScript but outputs nothing. Use for if/else, loops, variable declarations.
Output (unescaped)
<%- value %>
Outputs raw HTML — no escaping. Use only for trusted content like partials and HTML you control.
Comment
<%# comment %>
EJS comment — not sent to the browser at all. Different from an HTML comment which is visible in source.
⚠️

Never use <%- with user-supplied data. If a user enters <script>alert('XSS')</script> as their name and you render it unescaped, that script runs in every visitor's browser. Always use <%= for user data — it escapes dangerous characters automatically.

Quick reference in action

views/example.ejs
<!-- Output escaped value -->
<h1><%= user.name %></h1>

<!-- Execute JS (no output) -->
<% const greeting = user.isAdmin ? 'Admin' : 'User'; %>
<p>Role: <%= greeting %></p>

<!-- Output unescaped HTML (trusted content only) -->
<%- include('partials/header') %>

<!-- EJS comment — invisible in browser source -->
<%# This is a dev comment, never sent to browser %>
05
Passing Data to Templates
Everything the template needs must be passed from the route — nothing sneaks in
Key Concept

The second argument of res.render() is a plain JavaScript object. Every key in that object becomes a variable directly available inside the EJS template. If you pass { user, products, title }, you can use <%= user.name %>, <%= title %> in the template without any prefix.

route → template data flow
// Route — fetch and pass data
router.get('/dashboard', async (req, res) => {
  const user   = await User.findById(req.user.id);
  const orders = await Order.find({ userId: req.user.id });

  res.render('dashboard', {
    title: 'My Dashboard',     // ← becomes <%= title %>
    user,                        // ← becomes <%= user.name %>
    orders,                      // ← becomes <% orders.forEach %>
    orderCount: orders.length,  // ← becomes <%= orderCount %>
  });
});
views/dashboard.ejs — using the data
<head>
  <title><%= title %></title>   <!-- "My Dashboard" -->
</head>
<body>
  <h1>Welcome, <%= user.name %></h1>
  <p>You have <%= orderCount %> orders.</p>
</body>

res.locals — Global data for all templates

Sometimes you want certain data available in every template without passing it from every route — the logged-in user, the site name, flash messages. Use res.locals in a middleware for this.

middleware — global template variables
// Run before all routes
app.use((req, res, next) => {
  res.locals.currentUser = req.user || null;
  res.locals.siteName    = 'Sheryians Shop';
  res.locals.flashMsg    = req.session?.flash || null;
  next();
});

// Now every .ejs file can use:
// <%= currentUser?.name %>  <%= siteName %>  <%= flashMsg %>
// Without any route needing to pass them explicitly
06
Loops & Conditionals
Real JavaScript inside your HTML — if/else, forEach, ternary

Conditionals — if / else

views/navbar.ejs
<% if (currentUser) { %>
  <a href="/dashboard">My Dashboard</a>
  <a href="/logout">Logout</a>
<% } else { %>
  <a href="/login">Login</a>
  <a href="/register">Register</a>
<% } %>

<!-- Inline ternary for small things -->
<span class="badge <%= order.status === 'paid' ? 'green' : 'red' %>">
  <%= order.status %>
</span>

Loops — forEach over an array

views/products.ejs
<% if (products.length === 0) { %>
  <p>No products found.</p>
<% } else { %>
  <div class="grid">
    <% products.forEach(product => { %>
      <div class="card">
        <h3><%= product.name %></h3>
        <p>₹<%= product.price %></p>
        <a href="/products/<%= product._id %>">View</a>
      </div>
    <% }) %>
  </div>
<% } %>
💡

Pattern: Always check array.length before looping and show an empty state. Never assume data exists — if the DB returns an empty array and you loop without checking, the user just sees a blank page with no explanation.

07
Partials & Layouts
Reusable template fragments — write header and footer once, use everywhere
Key Concept

A partial is a reusable EJS fragment — a piece of a page you include in multiple templates. Instead of copying your <nav> into every page, you write it once in partials/header.ejs and include it with <%- include('partials/header') %>. Change it once, updates everywhere.

Create partials
views/partials/header.ejs
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title><%= title %> | <%= siteName %></title>
  <link rel="stylesheet" href="/style.css">
</head>
<body>
  <nav>
    <a href="/"><%= siteName %></a>
    <% if (currentUser) { %>
      <a href="/logout">Logout</a>
    <% } %>
  </nav>
views/partials/footer.ejs
  <footer>
    <p>© 2025 <%= siteName %></p>
  </footer>
</body>
</html>
Use partials in a page
views/dashboard.ejs — complete page
<%- include('partials/header') %>   <!-- unescaped — it's trusted HTML -->

<main>
  <h1>Welcome back, <%= currentUser.name %></h1>

  <% orders.forEach(order => { %>
    <div class="order-card">
      <p>Order #<%= order._id %> — ₹<%= order.total %></p>
    </div>
  <% }) %>
</main>

<%- include('partials/footer') %>

Passing data into a partial

You can pass extra variables specifically to a partial using a second argument in include():

Scoped partial data
<!-- Pass a specific product into the card partial -->
<% products.forEach(product => { %>
  <%- include('partials/product-card', { product }) %>
<% }) %>

<!-- partials/product-card.ejs can now use <%= product.name %> -->
08
EJS vs React — Pick the Right Tool
It's not either/or — different problems, different tools
FactorEJS (SSR)React (CSR/SPA)
Rendering location Server — full HTML sent to browser ✓ Client — browser builds the page from JS
SEO Excellent — content in HTML from the start ✓ Needs extra work (Next.js SSR) for good SEO
Setup complexity Zero — just npm install ejs ✓ Build step, bundler, state management needed
Interactivity Limited — needs separate JS for dynamic UI Excellent — React state, hooks, real-time updates ✓
API needed? No — data goes straight into the template ✓ Yes — React must fetch data from a separate API
Initial page load Fast — complete HTML, no JS needed to paint ✓ Slower — blank page until JS bundle downloads and runs
Best for Admin panels, emails, blogs, SEO pages ✓ Dashboards with complex state, real-time apps, SPAs
The Real-World Pattern

Many production apps use both. The public-facing marketing site and blog use EJS (fast, SEO-friendly). The logged-in app dashboard uses React (rich interactivity). You're not choosing one forever — you're choosing the right tool for each part of the product.

💡

EJS teaches you the foundation. Next.js, Remix, and Astro — the most popular React frameworks in 2025 — are all built on server-side rendering. When you understand how EJS renders a template on the server, you understand exactly what those frameworks are doing under the hood, just with a more sophisticated component model on top.
